# Excel インポートフォーマット定義書

## 概要

Dify DSL Builder にインポートする設定ファイル（Excel形式）のフォーマット仕様です。
本ファイルに業務ルールを記入してアップロードすると、4つの Dify ワークフローDSLファイルが自動生成されます。

**対象ユーザー**: 業務部門の担当者（エンジニア知識不要）

---

## Excelファイル構成

| シート名 | 必須 | 内容 | 記入量の目安 |
|---------|------|------|-------------|
| **①基本設定** | ◎ | 業務名・接続先など基本情報 | 5〜6項目 |
| **②チェック観点** | ◎ | AIがチェックする観点とOK/NGルール | 5〜15行 |
| **③差戻しコメント** | ◎ | NG時に申請者へ返すコメント文 | 10〜30行 |
| **④チェック項目** | ◎ | 結果記録に使うチェック項目名 | 5〜15行 |
| **⑤データ連携** | △ | 外部システムに送るデータ項目 | 3〜10行 |

> △ = 外部連携しない場合は空欄のままでOK。基本設定の接続先URLも空欄にしてください。

---

## 各シートの詳細

---

### シート① 「基本設定」

**レイアウト: 縦型（項目名 | 値 の2列）**

| 行 | A列: 項目名 | B列: 値（★入力欄） | 補足説明 |
|----|------------|-------------------|---------|
| 1 | 業務名 | 書面審査業務 | この業務の名前（自由記述） |
| 2 | 業務の説明 | 企業間の契約に際しての書面審査を自動化する業務 | 業務内容の簡単な説明（省略可） |
| 3 | 登録先システム名 | BPaaS 業務管理システム | 結果を登録するシステムの名前 |
| 4 | 接続先URL | https://example.com/api/process | 結果登録APIのURL（わからなければ空欄） |
| 5 | 通信方式 | POST | POST / GET / PUT のいずれか |

**ポイント:**
- 「業務名」は必須。DSLファイル名にも使われます
- 「接続先URL」は後から画面上で変更可能です
- AIモデル設定（モデル名・Temperature等）はテンプレートの推奨値が自動適用されます

**記述例:**

```
┌──────────────────┬──────────────────────────────────────────────┐
│ A列: 項目名       │ B列: 値                                      │
├──────────────────┼──────────────────────────────────────────────┤
│ 業務名            │ 注文書審査業務                                 │
│ 業務の説明         │ 注文書の受領から承認・差戻し判定までを自動化する業務  │
│ 登録先システム名    │ BPaaS 業務管理システム                          │
│ 接続先URL         │ https://external.rsi-bpaas-dev.rinfra.ricoh.com/process │
│ 通信方式          │ POST                                          │
└──────────────────┴──────────────────────────────────────────────┘
```

### シート② 「チェック観点」

**レイアウト: 横型テーブル（1行 = 1つのチェック観点）**

| 列 | ヘッダー名 | 型 | 必須 | 説明 |
|----|----------|-----|------|------|
| A | チェック観点名 | テキスト | ◎ | AIがチェックする項目の名前 |
| B | OK/NGルール | テキスト（複数行可） | ◎ | どういう場合にOK、どういう場合にNGか |
| C | ブロッキング | ○ or 空欄 | - | ○ = この項目がNGなら後続チェックを中止 |
| D | 有効 | ○ or 空欄 | - | 空欄 = 今回は使わない（将来用に残す） |

**ポイント:**
- **「OK/NGルール」が最も重要な列**です。AIはこのルールに従って書面をチェックします
- ルールは「〜の場合はOK」「〜の場合はNG」のように、**判定基準を明確に**書いてください
- 「ブロッキング」は書面の種類判定など、最初に確認すべき項目に○を付けます
- 「有効」列に何も書かないとその観点はスキップされます（デフォルトは有効）

**記述例:**

```
┌──────────────┬──────────────────────────────────────────────────┬──────────┬────┐
│ A: チェック観点名│ B: OK/NGルール                                     │ C: ﾌﾞﾛｯｷﾝｸﾞ│ D: 有効│
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 書面種別判定    │ 書面種別に"注文書"が含まれていなければNG。              │ ○        │ ○  │
│              │ 書面未添付の場合もNG。                                │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 契約日（注文日）│ 注文書の日付と突合データが一致していればOK。              │          │ ○  │
│              │ 日付が未記載の場合はNG。                               │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 受注企業名     │ 企業名が省略されず記載されており、突合データと            │          │ ○  │
│              │ 一致していればOK。不一致または未記載はNG。               │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 品目          │ 品目が突合データと一致していればOK。                     │          │ ○  │
│              │ 白物家電は販売不可のため、品目が白物家電の場合はNG。       │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 数量          │ 数量が突合データと一致していればOK。                     │          │ ○  │
│              │ 不一致または未記載はNG。                               │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 発注金額       │ 発注金額が突合データと一致していればOK。                  │          │ ○  │
│              │ 基本契約なし かつ 受注額100万円以上の場合はNG。           │          │    │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 押印          │ 取引先の押印があればOK。なければNG。                    │          │ ○  │
├──────────────┼──────────────────────────────────────────────────┼──────────┼────┤
│ 担当者名       │ 社判押印企業の担当者名がフルネームで記載されていれば       │          │ ○  │
│              │ OK。フルネームでなければNG。                           │          │    │
│              │ 社判を押印していない企業の担当者名はチェック不要。         │          │    │
└──────────────┴──────────────────────────────────────────────────┴──────────┴────┘
```

### シート③ 「差戻しコメント」

**レイアウト: 横型テーブル（1行 = 1つの差戻しコメント）**

| 列 | ヘッダー名 | 型 | 必須 | 説明 |
|----|----------|-----|------|------|
| A | No. | 数値 | ◎ | 連番（1, 2, 3...） |
| B | 差戻しコメント | テキスト | ◎ | NG時に申請者に通知するコメント文 |
| C | 分類 | 選択 | ◎ | **書面不備** or **データ不一致** |

**ポイント:**
- AIが書面チェック結果を見て、該当するコメントを自動選択します
- **コメント文はそのまま申請者に通知される**ので、丁寧な文面にしてください
- 「分類」は2種類:
  - **書面不備**: 書面そのものに問題がある場合（未添付、押印不備など）
  - **データ不一致**: 書面と社内データの不一致（金額、企業名、数量など）

**記述例（先頭5行）:**

```
┌────┬───────────────────────────────────────────────────────────┬──────────┐
│ A: No.│ B: 差戻しコメント                                          │ C: 分類   │
├────┼───────────────────────────────────────────────────────────┼──────────┤
│  1  │ 書面が未添付です。添付いただき、再申請して下さい。                │ 書面不備   │
│  2  │ 注文書の(控)・(仮)が添付されていますので、「手配可の差戻し」     │ 書面不備   │
│     │ とします。正式な注文書を入手次第、再申請して下さい。              │          │
│  3  │ 注文書に『注文日』が未記載です。再申請頂く際は記載頂いた日付と    │ ﾃﾞｰﾀ不一致 │
│     │ V-UPデータ契約締結日（注文日）が一致している事を確認して下さい。   │          │
│  4  │ 注文書の『お客様会社名』が未記載です。記載していただき、          │ ﾃﾞｰﾀ不一致 │
│     │ 再申請して下さい。                                           │          │
│  5  │ 『契約金額』が『注文書』と『V-UPデータ/契約金額』で不一致です。    │ 書面不備   │
└────┴───────────────────────────────────────────────────────────┴──────────┘
```

### シート④ 「チェック項目」

**レイアウト: 横型テーブル（1行 = 1つのチェック項目）**

| 列 | ヘッダー名 | 型 | 必須 | 説明 |
|----|----------|-----|------|------|
| A | No. | 数値 | ◎ | 連番（1, 2, 3...） |
| B | チェック項目名 | テキスト | ◎ | 審査結果として記録される項目名 |
| C | 分類 | 選択 | ◎ | **書面不備** or **データ不一致** |

**ポイント:**
- チェック項目は「どこに問題があったか」を分類するためのラベルです
- AIがチェック結果を見て、該当する項目を自動判定します
- 差戻しコメントとは別物です（コメント＝申請者への文面、チェック項目＝社内記録）

**記述例:**

```
┌────┬────────────────────────────────┬──────────┐
│ A: No.│ B: チェック項目名                  │ C: 分類   │
├────┼────────────────────────────────┼──────────┤
│  1  │ 書面未着：添付書面不備               │ 書面不備   │
│  2  │ 書面未着：表紙不備                  │ 書面不備   │
│  3  │ 書面：注文日-無/不備                │ 書面不備   │
│  4  │ 書面：甲)会社名（EU）-不備           │ 書面不備   │
│  5  │ 書面：甲)役職者名or担当者名-無       │ 書面不備   │
│  6  │ 書面：お客様契約印-不備              │ 書面不備   │
│  7  │ ﾃﾞｰﾀ不一致：契約金額(税抜)           │ ﾃﾞｰﾀ不一致 │
│  8  │ ﾃﾞｰﾀ不一致：甲)会社名(EU)           │ ﾃﾞｰﾀ不一致 │
│  9  │ ﾃﾞｰﾀ不一致：(明細)商品名             │ ﾃﾞｰﾀ不一致 │
│ 10  │ ﾃﾞｰﾀ不一致：数量                   │ ﾃﾞｰﾀ不一致 │
│ 11  │ ﾃﾞｰﾀ不一致：支払条件                │ ﾃﾞｰﾀ不一致 │
│ 12  │ 100万円以上：注文書受注不可          │ 書面不備   │
└────┴────────────────────────────────┴──────────┘
```

### シート⑤ 「データ連携」

**レイアウト: 横型テーブル（1行 = 1つのデータ項目）**

| 列 | ヘッダー名 | 型 | 必須 | 説明 |
|----|----------|-----|------|------|
| A | 項目名（英語） | テキスト | ◎ | システムに送る項目の名前（半角英数_） |
| B | 抽出ルール | テキスト | ◎ | AIがどうやってこの値を取り出すかの指示 |
| C | 必須 | ○ or 空欄 | - | ○ = この項目は必ず送る |

**ポイント:**
- 「項目名（英語）」は外部システムが受け取る際のキー名です（IT部門に確認してください）
- 「抽出ルール」はAIへの指示文です。日本語で「どの情報を取り出すか」を書けばOKです
- 外部連携しない場合、このシートは空のままで構いません

**記述例:**

```
┌──────────────┬────────────────────────────────────────────────────┬────┐
│ A: 項目名(英語) │ B: 抽出ルール                                        │ C: 必須│
├──────────────┼────────────────────────────────────────────────────┼────┤
│ company_name  │ 「契約者（甲）お客様名（書面用）」の値を取り出してください。  │ ○  │
│ system_status │ 審査結果が"承認"なら"completed"、"差戻し"なら"returned"。  │ ○  │
│ decision_reason│ 差戻し理由の文章。承認の場合は空欄。                      │    │
│ contract_number│ 「契約番号」の値。                                      │    │
│ case_number   │ 「案件番号」の値。存在しない場合は12345678。               │    │
└──────────────┴────────────────────────────────────────────────────┴────┘
```

## バリデーション仕様

インポート時に以下を自動チェックします:

| チェック内容 | エラー時の表示 |
|------------|-------------|
| シート①「業務名」が空 | 「業務名」は必須です |
| シート②が0行 | チェック観点を1つ以上入力してください |
| シート②「OK/NGルール」が空の行がある | 行N: OK/NGルールが未入力です |
| シート③「差戻しコメント」が空の行がある | 行N: コメント文が未入力です |
| シート③「分類」が不正値 | 行N: 分類は「書面不備」か「データ不一致」で入力してください |
| シート④「チェック項目名」が空の行がある | 行N: チェック項目名が未入力です |
| シート⑤「項目名（英語）」に全角文字 | 行N: 項目名は半角英数で入力してください |

## 技術仕様（開発者向け）

### シート名とマッピング

| シート名 | config プロパティ |
|---------|-----------------|
| ①基本設定 | name, desc, apiSystemName, apiUrl, apiMethod |
| ②チェック観点 | checkCriteria[] |
| ③差戻しコメント | returnComments[] |
| ④チェック項目 | checkItems[] |
| ⑤データ連携 | extractionParams[] |

### 分類値のマッピング

| Excel上の表記 | 内部値 |
|-------------|-------|
| 書面不備 | document_check |
| データ不一致 | data_verification |

### 読み取り開始位置

- 全シート共通: **2行目からデータ読み取り**（1行目はヘッダー）
- シート①のみ例外: A列=項目名、B列=値 の形式で1行目からデータ

### 使用ライブラリ

- SheetJS (xlsx.js) v0.20+ via CDN
- `XLSX.read(data, {type: 'array'})` でバイナリ読み取り
- `XLSX.utils.sheet_to_json(sheet, {header: 1})` で2次元配列化
